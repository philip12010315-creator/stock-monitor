## 核心問題與自動化方案抉擇 (已更新)
**最終決定：實作「半自動資料導入 + 全自動雲端同步」**
- **資料源**：維持每日由使用者手動從 XQ 匯出 CSV 檔案至 `發動結果` 資料夾，以確保資料精確度與 XQ 一致。
- **後續自動化**：Node.js 伺服器 (`server.js`) 已全面自動化。一旦偵測到新檔案，會自動執行：
    1. 重新解析數據並更新 `data.json`。
    2. 即時透過 WebSocket 推送到區域網路內的裝置。
    3. **自動調用 `同步到GitHub.bat`** 將最新數據與網頁程式碼推送到 GitHub Pages。
- **優點**：使用者只需做「匯出」一個動作，其餘流程（包含上傳網頁）皆由系統在背景自動完成。
