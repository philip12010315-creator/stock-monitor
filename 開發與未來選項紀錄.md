## 核心問題與自動化方案抉擇
**目標：** 希望能在每個開盤日的「晚上 7 點」自動將選股結果匯出成 CSV，並讓 Node.js 能夠推播。
**難點：** XQ 系統的「自動排程」和「資料匯出」屬於付費功能模組，且 XQ 強制倚賴 Windows 介面運行，不開電腦或不開軟體就無法運作。因此我們擬定了兩個繞過付費模組的替代方案：

### 方案一：保留 XQ，使用 Python 滑鼠畫面自動化 (UI Automation)
- **作法**：寫一支 Python 工具，每天晚上 19:00 使用 Windows 的工作排程器啟動。程式會強制將 XQ 拉到畫面最上層，然後模擬人類操作，自己去點擊「選股」「執行」「匯出到桌面」。
- **優點**：可以完美發揮 XQ 軟體的優勢，並且完全保留你花時間開發好的 `.xs` 腳本。
- **致命傷**：
  1. 每天 19:00 執行當下，你的電腦**不能處於睡眠或是螢幕鎖定的登出狀態**。
  2. 執行時不能遭到全螢幕遊戲覆蓋，滑鼠會被強制徵用約十幾秒的操作時間。
- **解方**：若不想動用到家裡電腦，只能每個月花費台幣 $500~$1000 租借一台永遠不關機的雲端 Windows 主機 (VPS) 來跑這支自動化程式。

### 方案二：捨棄 XQ 取代，開發純雲端 Python 直連爬蟲
- **作法**：廢棄 XQ 的腳本與資料庫。我們手寫一支新的 Python 爬蟲，每天 19:00 連線到「台灣證交所 / 櫃買中心」去抓取全市場收盤價與外資買賣超。我們用 Python 自己把 60 日的外資平均成本算出來，並篩選出 1.04 ~ 1.10 倍的股票。
- **優點**：這是**最為自由且免費的最終解法**。Python 程式可以放在 GitHub Actions 等免費雲端上每天時間一到在背景默默執行。**你的電腦就算關機沒電**，程式照樣生出名單。
- **致命傷**：
  1. 前期開發好的 XQ (`.xs`) 選股腳本將被廢棄不採用。
  2. 純以演算法推算的「外資 60 日成本」，可能會跟 XQ 自己公司內部經過特殊整理的數據在小數點上有微乎其微的些微誤差。
