# 🚀 成本推演模型 (Stock Monitor) - 新電腦環境建置指南

> **給未來的 Antigravity (AI助手) 的提示**：
> 哈嘍！當使用者請你閱讀這份檔案時，代表他正在一台全新的電腦上想要運行這個「外資成本監控系統」。你的任務是協助他無痛完成環境建置，確保這台電腦能像原本的開發機一樣順暢地更新與發布資料到 GitHub。

---

## 🛠️ 第一步：檢查與安裝基礎環境
請協助使用者確認並安裝以下必備軟體：

1. **Node.js (必要)**
   - 系統需要 Node.js 來執行 `server.js` (負責解析 CSV 並抓取 Yahoo Finance 現價)。
   - 請引導使用者下載並安裝 Node.js (建議 LTS 版本)。
   - 安裝後，請在終端機執行 `node -v` 確保安裝成功。
   - 執行 `npm install` 來安裝 `package.json` 中的相依套件 (例如 `csv-parser` 等)。

2. **Git 版本控制 (必要)**
   - 系統需要 Git 才能將更新後的網頁推送到 GitHub Pages。
   - 請確認新電腦是否有安裝 Git (`git --version`)。
   - **[重要]** 協助使用者完成 GitHub 帳號登入與授權，確保在這台電腦執行 `git push` 時不會遇到權限阻擋。

---

## 🔄 第二步：版本同步機制設定 (防衝突機制)

因為現在有**兩台以上的電腦**可能會更新同一個 GitHub 專案 (`philip12010315-creator/stock-monitor`)，為了防止版本衝突 (Merge Conflict)，我們必須修改同步腳本。

請未來的 Antigravity 協助檢查並優化目錄下的 `.bat` 腳本（特別是 `同步到GitHub.bat`），確保腳本的執行邏輯為：
1. `git stash` (暫存本地更動，如果有)
2. `git pull origin main` (先從遠端拉取最新的網頁與資料庫狀態)
3. `git stash pop` (還原本地更動)
4. `git add .` -> `git commit` -> `git push` (最後才將新的 CSV 更新推上去)

---

## 📈 第三步：XQ 全球贏家匯出設定

請引導使用者在新電腦的 XQ 系統中，重新掛載我們寫好的 XS 腳本（`外資成本溢價篩選.xs`）：
1. 在 XQ 中匯入並編譯該指標。
2. 確保「每日自動匯出 CSV」的路徑，精準對應到新電腦這個資料夾裡的 `data.csv` (或是 `server.js` 指定讀取的位置)。

---

## ✅ 第四步：啟動與測試

當上述都完成後，請引導使用者進行一次完整測試：
1. 確認 XQ 成功匯出最新 CSV。
2. 點擊執行 `server.js` (或相關啟動的 .bat) 確認能成功轉換出 JSON。
3. 點擊 `同步到GitHub.bat`，觀察終端機是否有成功 Push 到遠端的訊息。
4. 打開手機或瀏覽器，檢查 `https://philip12010315-creator.github.io/stock-monitor/` 是否成功更新。

完成以上四步，這台新電腦就正式成為完美的第二工作站了！
